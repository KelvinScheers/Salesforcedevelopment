7. Wat is het verschil tussen een inline en block elementen?


Een block-level element begint altijd op een nieuwe regel en neemt de volle breedte van het browservenster in. Het strekt zich uit naar links en rechts uit zo ver als het kan.
Een inline element start niet op een nieuwe regel en beslaat slechts zo veel breedte als het nodig heeft om wat erin staat weer te geven.

9.Sinds de komst van HTML5 zijn er elementen die betekenissen hebben. Wat wordt er bedoeld met HTML semantics? Waarom is het gebruik van semantics belangrijk?


Het legt de nadruk op de overeenkomst tussen de naam van het element en de inhoud ervan.
Aan de inhoud van de elementen wordt een betekenisvolle naam gegeven.bijvoorbeeld :

< header>
< section>

Betere toegankelijkheid.

10. Wat voor content zou je in de volgende elementen gebruiken?
Geef in je antwoord voor elk element een toelichting


  • < header> - dit element vertegenwoordigt een container voor inleidende inhoud of een reeks navigatielinks.
  • < footer> - dit element vertegenwoordigt het deel wat onderaan een webpagina te zien is. Deze HTML footer kan de contactgegevens bevatten of een sitemap, met of zonder interactieve elementen.
  • < main> - dit element vertegenwoordigt het hoofdgedeelte van de pagina, de inhoud waar de webpagina over gaat.
  • < nav> - dit element vertegenwoordigt een set van navigatie links.
  • < section> - Dit element beschrijft een sectie van de applicatie of van het document. Een sectie bevat een bij elkaar horende groep elementen met inhoud.
  • < aside> - In dit element plaats je elementen op de pagina die zijdelings bij een hoofdartikel of sectie horen.
  • < article - dit element wordt meestal gebruikt om informatie te bevatten die onafhankelijk kan worden verspreid van de rest van de site of applicatie waarin het voorkomt.
